# CueYori Screenshot Plan

Capture screenshots after the production build is installed on a simulator or device.

## Apple App Store

Recommended iPhone sequence:

1. Splash/loading brand moment.
2. Home dashboard with active session.
3. Large Next Cue card.
4. Active session unified timeline.
5. Dishes and stages list.
6. Add/Edit Dish flow.

Use App Store Connect's currently required device sizes when uploading. Keep the same visual story across each device size.

## Google Play

Recommended phone sequence:

1. Home dashboard.
2. Active cooking session.
3. Unified timeline.
4. Add/Edit Dish flow.
5. Completed cooking state.

Also prepare:

- Feature graphic.
- App icon.
- Short description and full description from `store/google/listing.json`.

## Visual Rules

- Use the final CueYori app icon and splash assets.
- Keep screenshots in light mode first.
- Add dark-mode screenshots only if the store listing needs more visual contrast.
- Avoid debug banners, simulator warning overlays, and incomplete TODO content.
- Use realistic dish names and stage timings.
